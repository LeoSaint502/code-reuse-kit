# code-library-save — 任务结束时自动存档到个人代码图书馆

> Agent 在任务结束时自动执行 git 操作，让 post-commit hook 完成代码提取入库。
> **用户不需要碰命令行。**

## 触发条件

在以下情况**必须**执行：
- Task 结束时，有**新创建或修改**的代码文件（.py/.js/.ts/.sh 等）
- 用户在对话中提出"保存代码"、"入库"、"存档"等类似要求

## 行为步骤

按顺序执行，每步成功再走下一步：

### 第 1 步：确定项目根目录

```bash
# 如果已经在项目目录里，直接用当前目录
# 如果不知道当前在哪里，先获取工作目录
pwd  # Unix/Mac
cd   # Windows
```

项目根目录 = 当前任务主要操作的目录（不要让用户确认，自己判断）。

### 第 2 步：初始化 git 仓库（如果需要）

```bash
# 检查是否已经是 git 仓库
git rev-parse --git-dir 2>/dev/null

# 如果不是，初始化
git init
```

### 第 3 步：添加所有文件

```bash
git add -A
```

### 第 4 步：提交代码

根据本次任务内容生成一个有意义的提交信息：

```bash
git commit -m "auto: <简短描述本次添加了什么功能>"
```

提交信息示例：
- `auto: 添加 PDF 文本提取函数`
- `auto: 实现用户认证中间件`
- `auto: 添加数据验证工具函数`
- `auto: 修复 CSV 解析编码问题`

用中文、简洁、说清楚新增/修改了什么。

### 第 5 步：完成

告诉用户一句："代码已自动存档到个人代码图书馆 ✓"

> **无需手动运行 extract_from_diff.py** — post-commit hook 会在 git commit 后自动完成代码提取入库。

## 什么时候跳过

以下情况**不要执行**存档：
- 只有文档/注释/README 修改，没有实质性代码
- 项目不是 git 仓库 且 明显是临时目录（如 /tmp/）
- 用户明确说"不用保存"、"测试用的"等
- 本次修改只有配置文件（.json/.yaml 等），没有函数/类定义
